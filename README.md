# ICMP Tunneling🚀

Проект представляет собой реализацию клиент-серверной архитектуры для скрытой передачи команд и получения их результатов через ICMP-туннель. 

Программа разработана на C++17 в образовательных целях для демонстрации работы с сырыми сетевыми сокетами (Raw Sockets), формирования кастомных пакетов на уровне L3 модели OSI и инкапсуляции данных в секцию данных ICMP-пакетов (тип `Echo Request` / `Echo Reply`).

## Особенности и функционал

- **Обход стандартных портов:** Данные передаются внутри протокола ICMP, что позволяет обходить базовые настройки файрволов, фильтрующих TCP/UDP трафик.
- **Двусторонняя связь:** Клиент отправляет команду, сервер выполняет её (через `popen`) и возвращает результат `stdout` обратно клиенту.
- **Умная фильтрация:** Пакеты помечаются уникальными магическими сигнатурами (`TNRQ` для запросов, `TNRS` для ответов), чтобы игнорировать фоновый ICMP-трафик ОС.
- **Шифрование полезной нагрузки:** Данные команд и ответов не передаются в открытом виде. Используется XOR-шифрование с секретным ключом, что делает невозможным чтение передаваемых команд при анализе дампа трафика (например, через Wireshark) без знания ключа.
- **Поддержка файлов:** Клиент может отправлять как короткие текстовые команды напрямую из аргументов, так и длинные скрипты/команды, считывая их из указанного файла.
- **Кроссплатформенная сборка:** Полностью автоматизированная сборка через CMake с автоподтягиванием Google Test.

## Архитектура проекта (Руководство программиста)

Проект разделен на три основных компонента:
1. **`tunnel_core` (Библиотека)** — содержит класс-утилиту `IcmpUtils` и классы исключений. Отвечает за вычисление контрольных сумм (RFC 1071), валидацию IP, обертку и извлечение полезной нагрузки, а также выполнение shell-команд.
2. **`icmp_server`** — демон, который слушает сырые сокеты. При получении пакета с сигнатурой `TNRQ`, он извлекает команду, исполняет её в системе и отправляет результат клиенту в пакете `Echo Reply` с сигнатурой `TNRS`.
3. **`icmp_client`** — утилита CLI для отправки команд. Формирует `Echo Request`, внедряет команду, отправляет серверу и ожидает ответный пакет с результатом.

Документация доступна в папке `docs/index.html`.

## Требования к системе

- **ОС:** Linux (обязательное требование для работы с `SOCK_RAW`).
- **Компилятор:** GCC или Clang с поддержкой C++17.
- **Сборка:** CMake 3.14 или выше.
- **Права:** Запуск исполняемых файлов требует прав **root** (через `sudo`), так как ОС запрещает обычным пользователям формировать сырые сетевые пакеты.

## Установка и сборка

1. Клонирование репозитория
  ```bash
  git clone <url_вашего_репозитория>
  cd ICMP_tunneling
  ```

2. Сборка с помощью СMake
  ```bash
  mkdir build && cd build
  cmake ..
  cmake --build .
  ```

После сборки в папке build появятся три исполняемых файла: icmp_server, icmp_client и tests.

## Сценарии использования (Руководство пользователя)

1. Запуск сервера
Сервер запускается на целевой машине, которой вы хотите управлять. Он не требует аргументов.
  `sudo ./icmp_server`
Вывод:
  `Запуск ICMP сервера. Ожидание пакетов...`

2. Отправка команды (Клиент)
На клиентской машине отправьте команду, указав IP-адрес сервера.
  `sudo ./icmp_client 127.0.0.1 "ls -la .."`
Вывод:
  ```bash
  Пакет с командой отправлен. Ожидание результата...
  Ответ сервера:
  total 12
  drwxr-xr-x. 1 user user 148 May 24 16:54 .
  drwxr-x---+ 1 user user 518 May 24 15:42 ..
  drwxr-xr-x. 1 user user 394 May 24 16:22 build
  -rw-r--r--. 1 user user 159 May 24 15:46 .clang-format
  -rw-r--r--. 1 user user 723 May 24 16:16 CMakeLists.txt
  drwxr-xr-x. 1 user user  24 May 24 16:16 docs
  drwxr-xr-x. 1 user user 146 May 24 16:54 .git
  -rw-r--r--. 1 user user  15 May 24 15:44 .gitignore
  drwxr-xr-x. 1 user user  30 May 24 16:16 include
  -rw-r--r--. 1 user user   0 May 24 16:54 README.md
  drwxr-xr-x. 1 user user  90 May 24 16:16 src
  drwxr-xr-x. 1 user user  18 May 24 16:16 tests
  ```

3. Отправка сложной команды из файла
Если команда слишком длинная или содержит сложные экранирования, её можно записать в файл.
  ```bash
  echo "cat /etc/os-release | grep PRETTY_NAME" > payload.txt
  sudo ./icmp_client 127.0.0.1 payload.txt
  ```
Вывод:
  ```bash
  Команда прочитана из файла: payload.txt
  Пакет с командой отправлен. Ожидание результата...
  Ответ сервера:
  PRETTY_NAME="Fedora Linux 44 (Forty Four)"
  ```

## Тестирование
Проект покрыт тестами с использованием Google Test. Тесты охватывают алгоритмы вычисления контрольных сумм, работу с сигнатурами, валидацию файлов и IP-адресов.
Для запуска тестов выполните в папке build:
  `ctest`
  или
  `./tests`
